## Назначение

Эта заметка агрегирует все маппинги "датчик -- параметр -- порог -- алерт -- действие" для IoT-системы CityFarm. Канонические runtime-значения порогов находятся в `backend/internal/alerting/thresholds.go`. Здесь задокументирована агрономическая основа порогов из vault-карточек параметров.

## Сенсоры без карточки параметра

- **YL-69 (soil moisture):** Backend имеет пороги `moisture` (20-80% warn, 10-90% crit), но в курсе нет данных по влажности субстрата для гидропоники. Пороги project-derived.
- **Relay (water pump):** Управляющий элемент на GPIO17 (active LOW), не датчик. Логика управления в firmware.
- **Camera (Xiaomi Mijia 360):** Визуальная диагностика растений через YOLO pipeline. Пороговых числовых значений нет -- классификация состояния по изображению.

## TODO для backend

1. **Добавить `light` threshold** -- рекомендация на основе vault: warn < 3 000 люкс для микрозелени (целевые 5 000 люкс), warn < 10 000 люкс для салата (целевые ~13 300 люкс).
2. **Добавить `co2` threshold** -- рекомендация на основе vault: warn > 1 500 ppm (оптимум 1 000--1 500 ppm), info при < 400 ppm.
3. **Расширить `ec` WarnHigh** -- текущее 1.6 мСм/см покрывает только рассаду и вегетацию. Для поздних фаз (цветение, финал) EC может достигать 2.5 мСм/см. Рассмотреть фазо-зависимые пороги.
4. **Пересмотреть `temperature` WarnHigh** -- 35 C значительно шире vault-оптимума (20-30 C дневная). Рассмотреть снижение до 30 C.
5. **Пересмотреть `humidity` WarnLow** -- 30% ниже любого vault-диапазона (минимум 40% для жилого помещения, 60% для фермы). Рассмотреть повышение до 40%.
